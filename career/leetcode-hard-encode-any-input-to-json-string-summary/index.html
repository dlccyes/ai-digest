    • Topics: Serialization, Depth-First Search, Hash Table, Graph, Design

    Problem gist

    The task is to write an encoder that accepts an arbitrary input value and returns a JSON string. The ordinary JSON problem is easy when the input is already a tree: recursively encode primitives, arrays, and objects.

    The hard part is the phrase “any type.” Real runtime values are not always trees. They can be object graphs with shared references and cycles. The example cycle can be built like this:

    a = []
b = {"name": a}
a.append(b)

    Now a contains b, and b["name"] points back to a. A naive recursive serializer will loop forever:

    list -> dict -> list -> dict -> ...

    So the real problem is not just formatting JSON. It is converting a possibly cyclic object graph into an acyclic JSON tree.

    Key idea

    JSON itself has no pointer or reference primitive. A raw JSON array or object cannot directly contain a cycle. To handle cycles correctly, the encoder needs a small reference protocol on top of JSON.

    A practical schema is:

    • the first time a non-primitive object is seen, emit it with a stable "$id"
    • if the same object is seen again, emit {"$ref": "<id>"}
    • register each container before recursively encoding its children

    That last point is the main correctness detail. If the encoder waits until after encoding children to mark an object as seen, a self-reference or cycle will still recurse forever.

    For the example above, one valid output is:

    {"$id":"1","$type":"list","items":[{"$id":"2","$type":"dict","entries":[["name",{"$ref":"1"}]]}]}

    This is valid JSON. It is not the same as ordinary JSON object syntax, because ordinary JSON cannot preserve that cycle. The "$id" and "$ref" fields are what make the graph representable.

    How to derive the solution

    Start with the normal recursive serializer:

    1. primitives become JSON primitives
    2. lists become arrays of encoded children
    3. dictionaries become encoded key-value entries

    Then add the graph layer:

    1. keep a hashmap from runtime object identity to assigned JSON reference ID
    2. only track compound values, because primitives do not need identity
    3. when a compound value is reached, check the hashmap first
    4. if it already exists, return a reference object
    5. otherwise assign an ID immediately, then encode its children

    Using object identity matters. Two different empty lists are equal by value, but they are not the same object. A cycle is about identity, not equality.

    Dictionaries are encoded as an entries array instead of a normal JSON object so the encoder can support non-string keys. Standard JSON object keys are strings, but Python dictionaries can use integers, tuples, enums, and other hashable objects as keys. The entries representation keeps the solution general.

    Python solution

    import base64
import json
import math
from collections.abc import Mapping
from typing import Any


class Solution:
    def encode(self, value: Any) -> str:
        return CycleSafeJsonEncoder().encode(value)


class CycleSafeJsonEncoder:
    def __init__(self) -> None:
        self._object_to_ref_id: dict[int, str] = {}
        self._next_ref_id = 1

    def encode(self, value: Any) -> str:
        normalized = self._normalize(value)
        return json.dumps(
            normalized,
            allow_nan=False,
            separators=(",", ":"),
        )

    def _normalize(self, value: Any) -> Any:
        if value is None or isinstance(value, (bool, int, str)):
            return value

        if isinstance(value, float):
            if math.isfinite(value):
                return value
            return {"$type": "float", "value": repr(value)}

        if isinstance(value, (bytes, bytearray)):
            return {
                "$type": type(value).__qualname__,
                "base64": base64.b64encode(bytes(value)).decode("ascii"),
            }

        if isinstance(value, Mapping):
            return self._normalize_mapping(value)

        if isinstance(value, (list, tuple)):
            return self._normalize_sequence(value)

        if isinstance(value, (set, frozenset)):
            return self._normalize_set(value)

        if hasattr(value, "__dict__") or self._has_slots(value):
            return self._normalize_object(value)

        return {
            "$type": type(value).__qualname__,
            "repr": repr(value),
        }

    def _normalize_mapping(self, value: Mapping[Any, Any]) -> Any:
        existing_ref = self._existing_ref(value)
        if existing_ref is not None:
            return existing_ref

        ref_id = self._remember(value)
        return {
            "$id": ref_id,
            "$type": type(value).__qualname__,
            "entries": [
                [self._normalize(key), self._normalize(item)]
                for key, item in value.items()
            ],
        }

    def _normalize_sequence(self, value: list[Any] | tuple[Any, ...]) -> Any:
        existing_ref = self._existing_ref(value)
        if existing_ref is not None:
            return existing_ref

        ref_id = self._remember(value)
        return {
            "$id": ref_id,
            "$type": type(value).__qualname__,
            "items": [self._normalize(item) for item in value],
        }

    def _normalize_set(self, value: set[Any] | frozenset[Any]) -> Any:
        existing_ref = self._existing_ref(value)
        if existing_ref is not None:
            return existing_ref

        ref_id = self._remember(value)
        return {
            "$id": ref_id,
            "$type": type(value).__qualname__,
            "items": [self._normalize(item) for item in value],
        }

    def _normalize_object(self, value: Any) -> Any:
        existing_ref = self._existing_ref(value)
        if existing_ref is not None:
            return existing_ref

        ref_id = self._remember(value)
        return {
            "$id": ref_id,
            "$type": f"{type(value).__module__}.{type(value).__qualname__}",
            "fields": [
                [name, self._normalize(field_value)]
                for name, field_value in self._object_fields(value).items()
            ],
        }

    def _existing_ref(self, value: Any) -> dict[str, str] | None:
        ref_id = self._object_to_ref_id.get(id(value))
        if ref_id is None:
            return None
        return {"$ref": ref_id}

    def _remember(self, value: Any) -> str:
        ref_id = str(self._next_ref_id)
        self._next_ref_id += 1
        self._object_to_ref_id[id(value)] = ref_id
        return ref_id

    def _object_fields(self, value: Any) -> dict[str, Any]:
        fields: dict[str, Any] = {}

        if hasattr(value, "__dict__"):
            fields.update(vars(value))

        for cls in type(value).__mro__:
            slots = getattr(cls, "__slots__", ())
            if isinstance(slots, str):
                slots = (slots,)
            for slot in slots:
                if slot in ("__dict__", "__weakref__") or slot in fields:
                    continue
                if hasattr(value, slot):
                    fields[slot] = getattr(value, slot)

        return fields

    def _has_slots(self, value: Any) -> bool:
        return any(hasattr(cls, "__slots__") for cls in type(value).__mro__)

    a = []
b = {"name": a}
a.append(b)

print(Solution().encode(a))

    Output:

    {"$id":"1","$type":"list","items":[{"$id":"2","$type":"dict","entries":[["name",{"$ref":"1"}]]}]}

    The runtime is O(n + s), where n is the number of reachable values and s is the size of the produced JSON. The extra memory is O(c) for the number of compound objects tracked in the identity map, plus the normalized output tree.

    Important edge cases

    The encoder must distinguish equality from identity. If the input is [[], []], those are two different lists and should receive different IDs. If the same list object appears twice, the second appearance should be a "$ref".

    Self-cycles need the same machinery as larger cycles. A list can contain itself:

    a = []
a.append(a)

    The encoder handles this because it remembers a before encoding a’s items.

    Special floating-point values also need deliberate handling. JSON does not support NaN, Infinity, or -Infinity, so the code above wraps them as tagged values instead of asking json.dumps to emit non-standard tokens.

    Interview follow-ups

    What if the interviewer wants ordinary JSON for acyclic inputs?

    Use two modes. In the first pass, detect whether the input graph has shared references or cycles. If it is a plain tree containing only JSON-compatible types, emit normal JSON. If a repeated object is found, switch to the tagged "$id" and "$ref" representation.

    The tradeoff is complexity. One-pass tagged encoding is simpler and always safe. Two-pass encoding gives prettier output for the common acyclic case, but it does more work and has two representations to test.

    What if the interviewer only asks to avoid infinite recursion?

    Then returning a placeholder such as "[Circular]" or null on repeated references is enough to terminate. That approach is common in logging and debugging because the goal is readability, not lossless reconstruction.

    The tradeoff is that it loses information. It cannot distinguish “this points back to the root” from “this points to some other earlier object.” The "$ref" approach is better when the output might be decoded later.

    How would decoding work?

    Decoding is the mirror image. First create empty shell containers for each object with a "$id", storing id -> object. Then fill in their items, entries, or fields recursively. Whenever a {"$ref": id} object appears, look up the already-created shell object.

    The two-phase construction is important for the same reason as the encoder’s early registration: a cycle can point to an object before all of that object’s children have been filled in.

    What if object types need to be reconstructed exactly?

    Then the schema needs a type registry. The encoder can write a module and class name, but the decoder should not blindly import and instantiate arbitrary classes from untrusted JSON. In production, the safer design is an allowlist mapping known type names to constructors.

    That changes the problem from pure serialization to protocol design. The interview point is to separate graph identity from type reconstruction: "$id" and "$ref" solve identity, while a registry solves type safety.

    Practical takeaway

    The fastest way to see this problem clearly is to stop thinking of the input as JSON-shaped. A cyclic value is a graph. JSON is a tree. The encoder’s job is to turn graph edges into explicit reference records so the final result is still valid JSON and the traversal always terminates.
